18   /**
19    * Creates a column that runs the folding RuleSuite.  This also forces registering the lambda functions used by that RuleSuite.
20    *
21    * FolderRunner runs all output expressions for matching rules in order of salience, the startingStruct is passed ot the first
22    * matching, the result passed to the second etc.  In contrast to ruleEngineRunner OutputExpressions should be lambdas with one parameter, that of the structure
23    *
24    * @param ruleSuite The ruleSuite with runOnPassProcessors
25    * @param startingStruct This struct is passed to the first matching rule, ideally you would use the spark dsl struct function to refer to existing columns
26    * @param compileEvals Should the rules be compiled out to interim objects - by default true
27    * @param debugMode When debugMode is enabled the resultDataType is wrapped in Array of (salience, result) pairs to ease debugging
28    * @param resolveWith This experimental parameter can take the DataFrame these rules will be added to and pre-resolve and optimise the sql expressions, see the documentation for details on when to and not to use this.
29    * @param variablesPerFunc Defaulting to 40 allows, in combination with variableFuncGroup allows customisation of handling the 64k jvm method size limitation when performing WholeStageCodeGen
30    * @param variableFuncGroup Defaulting to 20
31    * @param forceRunnerEval Defaulting to false, passing true forces a simplified partially interpreted evaluation (compileEvals must be false to get fully interpreted)
32    * @param forceTriggerEval Defaulting to true, passing true forces each trigger expression to be compiled (compileEvals) and used in place, false instead expands the trigger in-line giving possible performance boosts based on JIT.  Most testing has however shown this not to be the case hence the default, ymmv.
33    * @param useType In the case you must use select and can't use withColumn you may provide a type directly to stop the NPE
34    * @return A Column representing the QualityRules expression built from this ruleSuite
35    */

47         useType.getOrElse(dataRef.get())
