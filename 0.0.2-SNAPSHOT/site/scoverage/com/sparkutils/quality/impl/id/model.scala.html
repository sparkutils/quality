236 /**
237  * Represents a Guaranteed globally unique 160 bit ID within spark based on twitters 64bit snowflake id:
238  * leading 4 bits - type of id
239  * padded 4 bits for future use / ease of code
240  * next 48 bits - MAC address of the drivers host network card (unique for run with no central server / service requirement)
241  * next 32 bits - partition id
242  * next 41 bits - ms since 20210101
243  * padded 7 bits - for future use / ease of code
244  * remaining 24 bits - partition specific incremented row id - allowing 16777216 rows per ms, which is unlikely to occur
245  * and easy to manage overflow either way
246  *
247  * The organisation allows the base and first long to be fixed at the start of a partition reset with only the last long
248  * changing per row easing bit fiddling.  In the case of overflow the header remains untouched and only then must 41 bits ms value be
249  * re-evaluated, reducing clock slow downs (coincidentally around 41ms).
250  *
251  * Full time synchronisation across all clusters in a business is not required as MAC address provides driver uniqueness
252  * (assuming your cloud / network provider guarantees this uniqueness for routing).  Partition id provides segregation across
253  * a given action with timestamp ensuring repeated runs on the same cluster do not overlap.
254  */
