13    * Leverages the foldRunner to replace fields, the input fields are used to create a structure that the rules fold over.
14    * The fields are then dropped from the original Dataframe and added back from the resulting structure.
15    *
16    * NOTE: The field order and types of the original DF will be maintained only when maintainOrder is true.  As it requires access to the schema it may incur extra work.
17    *
18    * @param rules

20    * @param maintainOrder when true the schema is used to replace fields in the correct location, when false they are simply appended
21    * @param useType In the case you must use select and can't use withColumn you may provide a type directly to stop the NPE
22    * @return
23    */
